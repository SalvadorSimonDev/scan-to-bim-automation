¿Qué es un script de producción?

Un script de producción es código listo para ejecutarse con clientes y modelos reales. No es un prototipo ni un experimento — es el artefacto que automatiza trabajo en producción y del que depende la calidad del entregable.

CaracterísticaScript de pruebaScript de producción
Manejo de erroresMínimo o ningunotry/except + RollBack siempre
Transacciones RevitPuede omitirseObligatorio con cierre correcto
RendimientoNo importaOptimizado para >500 elementos
Logsprint()output.print_md() o archivo externo
UI bloqueanteTaskDialog libreCero ventanas modales en flujo principal
ValidacionesPocasVerifica parámetros antes de escribir
Colecciones masivasIteración directaFilteredElementCollector con filtros nativos
Regla de oro Toda escritura en el modelo Revit usa Transaction con try/except y RollBack en caso de fallo. Sin excepciones. Un script que corrompe el modelo en producción puede costar horas de trabajo del equipo.

Arquitectura general del sistema

El sistema conecta 4 fases con herramientas especializadas. La nube de puntos bruta entra por un extremo; el gemelo digital certificado en IFC 4.3 sale por el otro.

FASE 1 — Ingesta de Datos y Segmentación IA

01
Ingesta de Datos y Segmentación IA
Escaneo láser → clasificación automática por IA → geometría pre-modelada lista para Revit

El escaneo láser produce nubes de puntos masivas sin significado semántico. Las herramientas de IA de esta fase clasifican automáticamente cada punto: muros, suelos, vigas de acero, tuberías MEP, conductos HVAC. El resultado ya no es una nube ciega — es geometría que sabe lo que representa.

Muros, Suelos, Vigas,

de tuberías y conductos EW->>R: Familias nativas Revit (pipes/ducts) R->>R: Modelo preliminar listo

FASE 2 — Importación y Enlazado Automatizado en Revit

02
Importación y Enlazado Automatizado
Scripts pyRevit insertan la nube orientada y cargan el modelo IA con coordenadas compartidas correctas

El problema más frecuente al importar nubes de puntos en Revit es la desorientación del origen: si la nube no se alinea al Survey Point del proyecto, todo el modelo queda desplazado metros o kilómetros del sitio real. Esta fase lo automatiza.

sequenceDiagram participant U as Usuario BIM participant PY as Script pyRevit participant RV as Revit Document participant PC as Nube de Puntos (.rcp) participant AI as Modelo IA (.rvt) U->>PY: Ejecuta script desde panel pyRevit PY->>RV: Abre Transaction("Insertar_Nube") PY->>PC: Lee coordenadas de survey point PY->>RV: PointCloudInstance.Create() con transform orientado PY->>RV: Transaction.Commit() PY->>RV: Abre Transaction("Cargar_Modelo_IA") PY->>AI: Verifica ruta y versión del archivo PY->>RV: RevitLinkInstance.Create() PY->>RV: Aplica coordenadas compartidas PY->>RV: Transaction.Commit() RV->>U: Modelo + nube alineados PY->>U: output.print_md("Inserción completada")

FASE 3 — Auditoría QC y Control de Parámetros

03
Auditoría de Datos, Parámetros y QC
Escritura masiva de parámetros, detección de duplicados, validación de topología MEP y codificación Omniclass/Uniclass

Esta es la fase más crítica de automatización. Un modelo industrial puede tener miles de tuberías generadas por la IA con parámetros incompletos o elementos duplicados. El script de producción audita el modelo entero en segundos y escribe el estado de validación en cada elemento.

FASE 4 — Control de Desviaciones Scan-vs-BIM

04
Control de Desviaciones y Entrega
CheckToBuild / Verity compara el modelo final con la nube original para certificar precisión milimétrica

Antes de entregar, hay que demostrar cuantitativamente que el modelo BIM representa la realidad. Cada elemento recibe una desviación en milímetros. Si supera la tolerancia, se ajusta. Si no, queda certificado. Esta información viaja al IFC final.

sequenceDiagram participant RV as Modelo Revit participant CB as CheckToBuild / Verity participant SC as Nube de puntos original participant IFC as Exportador IFC 4.3 RV->>CB: Export modelo para análisis SC->>CB: Carga nube original CB->>CB: Algoritmo comparación punto a punto loop Por cada elemento BIM CB->>CB: Calcula desviación en mm alt Desviación menor a tolerancia CB->>RV: Escribe Desviacion_Scan_mm = X.X CB->>RV: Estado = CERTIFICADO else Desviación mayor o igual a tolerancia CB->>RV: Escribe Desviacion_Scan_mm = X.X CB->>RV: Estado = FUERA_DE_TOLERANCIA end end RV->>IFC: Exportar con parámetros de desviación IFC->>IFC: Validar schema IFC 4.3 IFC-->>Cliente: Gemelo digital certificado

Guía de instalación del entorno

Prerrequisitos

✅ Autodesk Revit 2024 o 2025
✅ pyRevit 4.8+       →  pyrevitlabs.io/download
✅ GitHub CLI (gh)    →  cli.github.com
✅ Python 3.x         →  para scripts externos de FASE 1
✅ Cuenta Aurivus AI o licencia EdgeWise
✅ Acceso CheckToBuild o plugin Verity

Paso a paso

  1. Clonar el repositorio 
    git clone https://github.com/SalvadorSimonDev/scan-to-bim-automation.git
cd scan-to-bim-automation
  2. Configurar pyRevit
    Abre pyRevit Settings desde la cinta de Revit → Custom Extensions → añade la ruta a scripts/ → recarga pyRevit. Los scripts aparecen automáticamente en el panel.
  3. Crear parámetros compartidos en Revit 
    Archivo → Parámetros compartidos → Nuevo archivo
Grupo: "BIM_Automatizacion"
Parámetros a crear:
  - Estado_Revision_IA    (Texto, instancia)
  - Codigo_Omniclass      (Texto, instancia)
  - Codigo_Uniclass       (Texto, instancia)
  - Fuente_IA             (Texto, instancia)
  - Desviacion_Scan_mm    (Número, instancia)
  4. Verificar variables de entorno (ya configuradas en .claude/settings.json) 
    {
  "BIM_STACK":      "Revit+pyRevit+Dynamo+CSharp",
  "BIM_IFC_TARGET": "IFC4.3",
  "BIM_AI_INGESTA": "Aurivus+EdgeWise",
  "BIM_QC_TOOL":    "CheckToBuild+Verity"
}
